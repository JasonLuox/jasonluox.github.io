[vue] vuex缓存

vue单页面应用刷新网页后vuex的state数据丢失的解决方案

Posted by Jason Luo on 2021-04-11

问题

用vuex做全局的状态管理时，刷新网页后，保存在vuex示例store里的数据会丢失

产生原因

store里的数据是保存在运行内存中的,当页面刷新时，页面会重新加载vue实例，store里面的数据就会被重新赋值。

解决思路

将state里的数据保存一份到本地存储(localStorage、sessionStorage、cookie）中

解决方案

  • localStorage
  • sessionStorage
  • cookie
  • 第三方包vuex-persistedstate

sessionStorage

在每次刷新页面之前将state中的数据存到sessionStorage当中去，刷新完成后再从sessionStorage中获取数据。

将以下代码放入app.vue中，可以保证每次刷新页面都可以触发

vuex-persistedstate

https://www.npmjs.com/package/vuex-persistedstate

安装/引入

npm install vuex-persistedstate --save

或

<script src="https://unpkg.com/vuex-persistedstate/dist/vuex-persistedstate.umd.js"></script>

使用

vuex-persistedstate提供三种存储方式 (localStorage、sessionStorage、cookie) 默认localStorage。

根据目录结构在store下的index.js文件下引入

vuex-persistedstate 3.x (for Vuex 3 and Vue 2)

vuex-persistedstate 4.x (for Vuex 4 and Vue 3)

sessionStorage
cookie

npm install js-cookie --save

使用

API

createPersistedState([options])

使用给定的选项创建插件的新实例。可以提供以下选项来配置插件以满足您的特定需求：

  • key <String>: 存储持久化state的秘钥，默认为vuex

  • paths <Array>： 可以部分保留状态的所有路径的数组。如果未给出路径，则完整状态将保留。如果给出一个空数组，则不会保留任何状态。必须使用点符号指定路径。如果使用模块，请包含模块名称。例如：“ auth.user”默认为未定义。

  • reducer <Function>：将根据给定路径调用以减少状态持久化的函数。默认值包括这些值。

  • subscriber <Function>: 用来设置mutation的订阅函数。默认为store => handler => store.subscribe(handler)

  • storage <Object>: 代替getState和setState（或与之组合）。默认为localStorage。

  • getState <Function>: 被调用以恢复state到之前的状态，默认使用storage

  • setState <Function>: 被调用以保持给定的state不变，默认使用storage

  • filter <Function>: 调用后用以过滤所有将最终触发存储状态的setState，默认为() => true

  • overwrite <Boolean>: 当触发将保存的值应用到state中时，判断是否直接用getState的输出覆盖现有状态，而不是将两个对象与deepmerge合并,默认为false

  • arrayMerger <Function>: 将保存的值应用到state中时用于合并数组，默认为function(store, saved){return saved}（保存状态替换提供的状态)

  • rehydrated <Function>: 当将保存的值应用到state中完成后将被调用的函数。当您使用Nuxt.js时，该功能非常有用，持久化状态的恢复是异步发生的。默认为store => {}

  • fetchBeforeUse <Boolean>: 一个布尔值，指示是否应在使用插件之前从存储中获取状态。默认为false

  • assertStorage <Function>: 插件初始化时触发的可重写功能，用于确保存储可用。默认情况下，是在给定的Storage实例上执行Write-Delete操作。请注意，默认行为可能会引发错误（例如DOMException：QuotaExceededError）
